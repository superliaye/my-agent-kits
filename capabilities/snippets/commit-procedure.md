# Commit Best Practices

1. Review `git status` and the relevant diffs, including staged changes, before committing.
2. Write a concise commit message that captures the purpose of the change at the appropriate abstraction level. Summarize the behavior or intent, not implementation details already obvious from the code.
3. When the change spans multiple aspects, mention only the most important themes, such as the overall feature, core rule changes, or notable scenarios covered.
