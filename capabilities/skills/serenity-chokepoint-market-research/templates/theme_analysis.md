# Chokepoint Theme Analysis — Working Template

> Portable research template for any supply-chain / value-chain theme (not only AI).
> Fill every section. Where you lack evidence, write `UNKNOWN — needs source` rather than guessing.
> Tag every line as **[F]** fact (with citation), **[I]** inference, or **[S]** speculation.
> This is a research framework, not investment advice.

## 4. Value-chain diagram
Map from end demand backward to the binding constraint. Replace layers as the theme requires.
```
End demand
  -> system / architecture
  -> subsystem
  -> component
  -> material / input
  -> manufacturing equipment
  -> testing & qualification
  -> logistics / transport
  -> energy / power
  -> financing / capital
```
Annotate each layer with: who supplies it, rough concentration, and whether it is a candidate constraint.

## 5. Candidate chokepoints
List each layer that could be the binding constraint and why. For each: indispensability, supply concentration, expansion latency, switching friction.

## 6. Chokepoint scorecard
Score each candidate layer 0–5 (0 = not a constraint, 5 = severe). Do not treat the total as a buy signal.

## 15. Falsifiers
Answer each explicitly:
- What evidence would prove this is NOT a bottleneck?
- How fast can competitors add capacity?
- Could customers redesign around the component?
- Could pricing normalize before earnings materialize?
- Is demand double-counted across customer announcements?
- Is the optimistic case already priced in?
- Are margins sustainable?
- Could geopolitical / regulatory / financing constraints reverse the thesis?
- Which assumptions rely on management guidance rather than independent evidence?
- What would make you stop researching or downgrade this?
